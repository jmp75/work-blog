Background

Note

air2stream is being ported to Python or perhaps Julia. The work described here may be obsolete by the time you read.

This is a follow up on Wrapping air2stream to access it from Python - part 2. In part 2 we found that f90wrap is doing a good job of wrapping the code of air2stream as it is.

In this post we will focus on providing an API to set inputs, parameters and retrieve outputs in memory rather than through files.

Installation

The Solara install instructions at the time of writing has instructions using a not conda. I usually prefer working from conda environments, but the first thing I learned was that setting up solara on top of conda did not pan out. Basically, because of some documentary dependencies like vaex ended up being needed, and a whole can of worm of incompatible package versions was opened.

So, I advise you stick with creating a virtual environment with venv as per the solara documentation.

python --version # 3.9.13; debian linux
cd $HOME/.venvs/
python -m venv solara-env
source ./solara-env/bin/activate
pip install solara # installs 1.32.0
~/.venvs/solara-env/bin/python -m pip install --upgrade pip
pip install matplotlib pandas vaex plotly

FranciscoGS/solara-geospatialDashboard on HuggingFace using a display function to show the leaflet map. A curious thing reading the code statically is that there is no import statement that could import this display function from a particular module.

https://solara.dev/documentation/getting_started/introduction has a similar use of a display function that appears not imported.

def build_map():
    x:xr.Dataset = create_dataset()
    gv = GeoViewer(x, lat="lat", lon="lon", identifier="station")
    map = gv.build_map(zoom=9)
    return map

@solara.component
def MyMapComponent():
    # Note the explicit import of `display` from `solara` seems to be superfluous, but works.
    # perhaps even without it, the file's `import solara` does magic
    from solara import display
    map = build_map()
    display(map)
