Background

I’ve had a (second hand) question from a user whether it was possible to write a custom “node model” for a C++ software stack for streamflow forecasting. It is a stack with a core in C++, but accessible via a C API by users from R, Matlab, Python and so on.

Though the use case deserves some more elicitation, this is worth exploring the feasible technical options based on this current and previous requests.

C++ plugin

https://hackaday.com/2018/07/12/its-all-in-the-libs-building-a-plugin-system-using-dynamic-loading/

The context is needing to implement a rainfall runoff model usable from swift but whose implementation already exists without dependency on swift.

A likely candidate for the plugin system is Boost.DLL.

Gave a trial implementation by starting to refactor software interfaces out of RunoffModel. This quickly led to difficulties if I tried to replicate something that intuitively works in e.g. C#, but is difficult to replicate in c++. Adding an ITimeStepModel interface introduces on top of that a diamond inheritance problem; as I read virtual inheritance is supposed to overcome this issue but I cannot get the behavior I want. The added complexity even in this relatively small trial does not bode well; will have to reconsider the approach.
